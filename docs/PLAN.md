# 기타 지판 시각화 학습 도구 — 기획안

> 프로젝트 코드네임: **i-wanna-be-john-mayer**
> 작성일: 2026-07-16 · 용도: 개인 학습용 · 상태: 인터뷰 기반 의도 확정 완료

---

## 1. 배경과 문제 정의

스케일 폼 자체는 손에 익었지만, 솔로가 "스케일 오르내림"처럼 들리는 단계에 머물러 있다.
원인은 두 가지로 정리된다.

1. **코드톤 타겟팅 감각 부재** — 지금 울리는 코드의 1·3·5·7이 지판 어디에 있는지
   즉시 떠오르지 않아서, 프레이즈를 "착지점" 중심으로 설계하지 못한다.
2. **지판 음이름 미암기** — "5번 줄 7프렛이 무슨 음?"에 바로 답하지 못해서,
   이론 지식(코드 공식, 인터벌)이 지판 위 실제 위치로 연결되지 않는다.

이 두 문제를 시각 자료 + 능동 훈련으로 해결하는 개인용 웹 도구를 만든다.

## 2. 확정된 의도 (인터뷰 결과)

| 항목 | 내용 |
|---|---|
| Outcome | 스케일과 **단일 코드**의 코드톤을 지판 전체에서 시각화하고, 음이름 상시 표시 + 퀴즈로 암기를 보조하는 웹앱 |
| User | 본인 (존 메이어 스타일 지향 기타리스트 · 개발자) |
| Why now | 코드톤 타겟팅 감각과 지판 음이름 암기가 안 잡혀 있어서 |
| Success | 연습 세션에서 실제로 옆에 띄워놓고 **계속 쓰게 되는 것** |
| Constraint | Next.js + TypeScript, 백엔드 없는 클라이언트 사이드, 반응형(PC·태블릿) |
| Out of scope (v1) | 코드 진행 시퀀서, 백킹트랙/오디오 재생, 계정·서버·동기화, 네이티브 모바일 앱 |

> 사용 장면은 "참조 + 탐색 + 훈련" 모두이지만, v1의 중심축은
> **① 단일 코드의 코드톤 시각화**와 **② 음이름 표시 + 퀴즈**다.

## 3. 핵심 사용자 시나리오

### 시나리오 A — 연습 중 참조 (기타를 들고)
E7 위에서 솔로 연습을 하다가 "3음(G#)이 어디였지?" 싶을 때,
화면에서 루트 `E` + 코드 타입 `7`을 선택하면 지판 전체에 코드톤이 도수별 색상으로 표시된다.
멀리서도 보이도록 노트가 크고 대비가 뚜렷해야 하며, 원하는 뷰까지 **3클릭 이내**여야 한다.

### 시나리오 B — 이론 탐색 (기타 없이)
"A 마이너 펜타토닉 박스 2와 CAGED C폼이 어떻게 겹치지?" 같은 궁금증이 생기면
키·스케일·포지션을 바꿔가며 지판 전체 뷰와 포지션별 뷰를 오가며 관계를 눈으로 확인한다.

### 시나리오 C — 암기 훈련 (하루 5분)
퀴즈 모드에 들어가 "이 위치의 음이름은?" 또는 "지판에서 모든 C를 클릭"을 반복한다.
처음엔 6·5번 줄만으로 범위를 좁혀 시작하고, 정답률이 오르면 범위를 넓힌다.
통계(정답률·응답 속도)는 localStorage에 쌓여 성장이 보인다.

## 4. 기능 명세

### v1 (MVP)

**F1. 지판 렌더러** — 모든 기능의 토대
- SVG 기반, 6현 × 22프렛, 표준 튜닝(EADGBE) 고정
- 너트, 프렛 번호, 인레이 마커(3·5·7·9·12·15·17·19·21) 표현
- 노트 라벨 3단 토글: **음이름 / 도수(1·b3·5…) / 숨김**
- 루트 음 강조 (모양 또는 테두리로 구분 — 색맹 대비 색상+형태 이중 부호화)
- viewBox 기반 반응형: 데스크톱·태블릿 가로에서는 풀 지판,
  좁은 화면에서는 가로 스크롤 또는 프렛 범위 축소

**F2. 스케일 뷰어**
- 키: 12키 전체
- 스케일: 메이저, 내추럴 마이너, 마이너 펜타토닉, 메이저 펜타토닉, 블루스
- 지판 전체 표시 + **포지션 필터**: 펜타토닉 5박스 / CAGED 5폼 단위로 하이라이트
- 키에 맞는 표기법 자동 선택 (F키면 A#이 아니라 Bb)

**F3. 코드톤 뷰어** — v1의 중심 기능
- 루트(12키) × 코드 타입: `maj` `m` `7` `maj7` `m7` (v1은 이 5개, 확장 여지 열어둠)
- 지판 전체에 코드톤 표시, **도수별 색상 구분** (1=루트, 3, 5, 7 각각 다른 색)
- 3음(장/단 결정음)과 7음(코드 성격 결정음)이 시각적으로 가장 잘 띄게

**F4. 퀴즈 모드 (음이름 암기)**
- 모드 A "이 음은?": 지판 위 한 위치가 하이라이트 → 음이름 4지선다
- 모드 B "모두 찾기": "지판에서 모든 C를 클릭하세요" → 클릭 판정
- 출제 범위 설정: 현 선택(예: 6·5번 줄만), 프렛 범위(예: 0~12)
- 세션 결과 + 누적 통계(정답률, 평균 응답 시간)를 localStorage에 저장

**F5. 상태 공유성**
- 현재 뷰 상태를 URL 쿼리에 인코딩 (`?mode=chord&root=E&type=7`)
  → 북마크해두면 연습 때 원클릭으로 복귀. 개인용 도구에서 가장 싼 "저장" 기능.

### v2+ 백로그 (v1에서 의도적으로 제외)

| 후보 | 메모 |
|---|---|
| 오버레이 모드 | 스케일(흐리게) + 코드톤(진하게) 동시 표시 — "안전한 착지점" 시각화. 코드톤 타겟팅 학습의 다음 단계로 가장 유력한 v2 기능 |

| 인터벌·코드톤 퀴즈 | 음이름 퀴즈의 확장판 |
| 커스텀 튜닝 · 카포 · 왼손잡이 뷰 · 24프렛 | 설정 확장 |
| 다크 모드 | 어두운 연습 환경 대응 |

## 5. 화면 구성 (UX)

```
┌─────────────────────────────────────────────────┐
│  [스케일] [코드톤] [퀴즈]          ← 모드 탭     │
│  키/루트 ▾   스케일·코드타입 ▾   라벨: 음이름 ▾  │  ← 컨트롤 바
├─────────────────────────────────────────────────┤
│                                                 │
│   ●──●──────●──●──  ← 지판 (화면의 주인공,      │
│   ──●──●──●──────    항상 최대 크기로)          │
│   ●──────●──●──●──                              │
│                                                 │
├─────────────────────────────────────────────────┤
│  포지션 필터: [전체] [1] [2] [3] [4] [5]        │
└─────────────────────────────────────────────────┘
```

설계 원칙:
- **지판이 주인공** — 컨트롤은 상단 한 줄로 최소화, 지판은 항상 최대 면적
- **3클릭 룰** — 어떤 뷰든 모드 탭 → 루트 → 타입, 3번의 조작 안에 도달
- **연습실 가독성** — 기타를 들고 1m 떨어져서도 읽히는 노트 크기와 색 대비
- 반응형: 데스크톱/태블릿 가로 = 풀 지판, 태블릿 세로·모바일 = 가로 스크롤 지판

## 6. 기술 설계

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | Next.js (App Router) + TypeScript | 사용자 선호. 페이지는 사실상 1개, 클라이언트 컴포넌트 중심 |
| 렌더링 | SVG (viewBox) | 지판·노트를 선언적으로 그리기 좋고 반응형이 공짜 |
| 이론 엔진 | 순수 TS 함수로 자체 구현 | pitch class(0–11) 연산 + 스케일/코드 인터벌 테이블이면 충분. 학습 겸 의존성 최소화. (막히면 `tonal` 라이브러리로 대체 가능) |
| 상태 | URL 쿼리 + React state | 뷰 상태는 URL에, 퀴즈 진행 등 휘발 상태는 컴포넌트에 |
| 영속화 | localStorage | 퀴즈 통계·설정. 서버 없음 |
| 테스트 | Vitest | 이론 엔진(음 계산, 스케일/코드 생성)은 유닛 테스트 필수 — 여기가 틀리면 도구 전체가 거짓말을 함 |
| 배포 | Vercel 또는 정적 export | 개인용이라 어느 쪽이든 무방 |

### 도메인 모델 스케치

```ts
type PitchClass = 0 | 1 | ... | 11;          // C=0 … B=11
type FretPos = { string: 1|2|3|4|5|6; fret: number };

const TUNING: PitchClass[] = [4, 9, 2, 7, 11, 4]; // E A D G B E (6번줄→1번줄)

// 핵심 연산: 위치 → 음
pitchAt(pos: FretPos): PitchClass

// 공식 테이블 (루트로부터의 반음 간격)
SCALES = { major: [0,2,4,5,7,9,11], minorPentatonic: [0,3,5,7,10], ... }
CHORDS = { maj: [0,4,7], m: [0,3,7], "7": [0,4,7,10], ... }

// 표기: 키 컨텍스트에 따라 C# vs Db 선택
spell(pc: PitchClass, keyContext: Key): string
```

이 엔진이 정확하면 뷰는 "지판의 모든 (string, fret)을 순회하며
현재 스케일/코드 집합에 속하는 음만 칠하는" 단순한 매핑이 된다.

## 7. 마일스톤

각 마일스톤은 그 자체로 **연습에 쓸 수 있는 상태**로 끝난다.

- **M1 — 지판 + 스케일 뷰어**: 이론 엔진(+테스트), 지판 렌더러, 키/스케일 선택, 라벨 토글.
  → 이 시점부터 이미 연습 참조용으로 사용 시작 가능
- **M2 — 코드톤 뷰어**: 루트×코드타입 선택, 도수별 색상, 포지션 필터.
  → 핵심 문제(코드톤 타겟팅)에 대한 답이 나오는 시점
- **M3 — 퀴즈 모드**: 두 가지 퀴즈, 범위 설정, localStorage 통계.
- **M4 — 폴리시**: URL 상태 인코딩, 반응형 다듬기, 접근성(색+형태 이중 부호화) 점검.

## 8. 성공 지표

정량 지표를 억지로 만들기보다, 개인 도구답게 정직한 기준을 쓴다.

1. **생존 테스트**: 완성 후 2주간의 연습 세션에서 실제로 이 도구를 띄우는가
2. **퀴즈 추이**: 음이름 퀴즈 정답률·응답 속도가 주 단위로 개선되는가
3. **체감 테스트**: 블루스 반주 위에서 코드 바뀔 때 3음·7음으로 착지하는 프레이즈를
   의식적으로 만들 수 있게 되었는가
