# UI/UX 재설계 디자인 스펙 — 현대적 프론트엔드 수준의 유려한 다듬기

- 날짜: 2026-07-16
- 상태: 사용자 승인 완료 (섹션별 승인)
- 범위: UX 구조 재설계 + 라이트/다크 듀얼 테마 + Tailwind CSS v4 전환
- 대상: 데스크톱 우선, 모바일 대응
- 기능: 기존 4개 모드(스케일/코드톤/오버레이/퀴즈) 동작 유지 + 소형 UX 기능 추가 허용

## 1. 목표와 비목표

### 목표
- 기능 중심의 소박한 UI를 현대 웹앱 수준의 완성도로 재설계한다.
- 라이트/다크 테마를 동등하게 지원한다(시스템 추종 + 수동 전환).
- Tailwind CSS v4로 스타일링 스택을 전환하되, 지판 SVG의 CSS 변수 체계는 유지한다.
- 기존 로직(`src/theory`, `src/quiz`, `src/audio`, `src/lib`)과 테스트 계약(`data-testid`, `aria-*`, role)은 변경하지 않는다.

### 비목표
- 새 학습 모드/이론 기능 추가 없음.
- 컴포넌트 라이브러리, 모션 라이브러리 도입 없음.
- 지판 SVG 렌더링 구조 변경 없음(시각 토큰과 소형 애니메이션 클래스만).

## 2. 정보 구조 & 레이아웃 (승인됨)

레이아웃 구조는 **스테이지 중심 단일 컬럼**(A안)이다. 위에서 아래로:

### 2.1 스티키 헤더 (앱 바)
- 상단 고정(sticky), 반투명 배경 + 백드롭 블러.
- 좌측: 앱 아이덴티티 — 기타 픽 모양 아이콘 + "Fretboard" 워드마크.
- 중앙: 모드 탭 4개(스케일 · 코드톤 · 오버레이 · 퀴즈). 기존 세그먼트 버튼을 탭으로 승격, 선택 탭에 슬라이딩 인디케이터.
- 우측: 사운드 토글 + 테마 토글(아이콘 버튼).

### 2.2 컨트롤 바 (컨텍스트 영역)
- 헤더 아래, 지판 위. 모드에 따라 내용이 바뀌는 한 줄 카드.
- 셀렉트는 라벨이 위에 붙는 필드 스타일. 세그먼트 그룹(라벨 표시, 포지션)은 필(pill) 그룹으로 통일.
- 퀴즈 설정(출제 현/퀴즈 종류/프렛 범위/시작 버튼)도 이 위치로 이동해 다른 모드와 위계를 통일한다.
- 모드 전환 시 높이 고정(기존 min-height 점프 방지 로직 계승). ~~내용 짧은 페이드 전환~~ (2026-07-17 사용자 피드백: 페이드가 깜빡임으로 인지되어 제거 — 리마운트 없이 내용만 교체).

### 2.3 스테이지 (지판 카드)
- 카드 헤더: 뷰 타이틀(예: "A 마이너 펜타토닉 — 박스 1") 좌측, **범례** 우측.
  - 범례(신규): 현재 모드에 맞는 색상 설명 — 스케일 모드: 루트/스케일음, 코드톤·오버레이 모드: 루트(1)/3/5/7도, 오버레이는 딤 처리된 스케일음 포함.
- 카드 본문: 지판 SVG 전체 폭.
- 모바일: 가로 스크롤 유지(min-width 900px) + 좌우 에지 페이드로 스크롤 가능함을 암시.

### 2.4 퀴즈 모드
- 지판 아래: 선택지 버튼을 문제 카드 스타일로 확대. 정답/오답 피드백은 색 + 아이콘 + 짧은 모션.
- 세션/누적 통계는 미니 스탯 칩(이번 세션, 누적 정답률, 평균 시간)으로 표시.

### 2.5 모바일 대응 수준
- 헤더: 모드 탭 4개 유지(축약 없음), 컨트롤 바는 2줄 랩핑.
- main 패딩 축소, 카드 라운드/여백 축소.

## 3. 디자인 시스템 (승인됨)

### 3.1 Tailwind v4 (CSS-first)
- `tailwind.config.*` 없이 `app/globals.css`의 `@theme`로 토큰 정의.
- 지판 SVG가 참조하는 CSS 변수(`--fb-*`, `--note-*`, `--tone-*`, `--mark-*`)는 이름 호환을 유지하고 라이트/다크 두 벌로 정의한다. SVG 코드는 색 변경 없이 테마를 자동 추종.

### 3.2 컬러
- **라이트**: 따뜻한 종이 톤 계승 — 배경 `#faf8f4` 계열, 표면 순백, 웜 그레이 라인. 대비 정돈.
- **다크**: 웜 차콜(`#1c1a17` 계열) 배경 + 한 단계 밝은 표면(`--surface`). 노트 컬러는 채도/명도를 올린 다크 변형으로 재조정 — 흰 라벨과 WCAG AA 대비 확보.
- 시맨틱 토큰: `--bg / --surface / --surface-2 / --line / --ink / --ink-muted / --accent` + 기존 지판·노트·퀴즈 마크 토큰.

### 3.3 테마 전환
- 3단 순환 토글: 시스템 → 라이트 → 다크. 사용자 *선호*(`system|light|dark`)는 `localStorage`(키: `theme`)에 저장한다.
- 적용 방식: `<html data-theme>`에는 항상 **해석된 값**(`light` 또는 `dark`)만 설정한다. 선호가 `system`이면 `prefers-color-scheme`로 해석하고, `matchMedia` change 이벤트로 실시간 갱신한다(선호가 `system`일 때만).
- FOUC 방지: `app/layout.tsx` head에 페인트 전 실행되는 인라인 스크립트가 저장된 선호를 읽어 해석된 `data-theme`을 설정.
- Tailwind `dark:` 변형은 `@custom-variant dark ([data-theme="dark"] &)`로 연결 — 해석된 속성만 보면 되므로 미디어 쿼리 분기가 불필요.

### 3.4 타이포그래피
- 시스템 폰트 스택 유지(한글 포함 최적). 숫자에 `tabular-nums`(프렛 번호, 통계).
- 위계: 워드마크 15px/semibold, 뷰 타이틀 17px/semibold, 본문 14px, 보조 12.5px.

### 3.5 형태 & 깊이
- 라운드: 카드 16px, 컨트롤 10px, 필 버튼 full.
- 라이트: 옅은 이중 그림자. 다크: 그림자 대신 1px 보더 + 표면 명도 차.

### 3.6 모션
- 기본 150–200ms ease-out. 모드 탭 슬라이딩 인디케이터. (컨트롤 바 내용 페이드는 2026-07-17 사용자 피드백으로 제거.)
- 노트 클릭: 짧은 펄스(사운드와 시각 동기). 퀴즈 정답: 팝 스케일, 오답: 미세 셰이크.
- `prefers-reduced-motion: reduce` 시 전부 비활성.

## 4. 구현 계획 (승인됨)

### 4.1 의존성
- 추가: `tailwindcss` v4, `@tailwindcss/postcss` (devDependencies). 그 외 없음.

### 4.2 파일 단위 변경
| 파일 | 변경 |
|---|---|
| `app/globals.css` | `@theme` 토큰 + 라이트/다크 변수 + SVG 전용 소량 CSS만 잔존, 나머지는 Tailwind 유틸리티로 이전 |
| `app/layout.tsx` | FOUC 방지 인라인 테마 스크립트 |
| `app/page.tsx` | 새 레이아웃 구조(헤더/컨트롤 바/스테이지) 반영 |
| `src/components/Controls.tsx` | 탭/필드/필 그룹 스타일 전환 |
| `src/components/Quiz.tsx` | 설정 이동, 선택지/피드백/스탯 칩 재구성 |
| `src/components/SoundToggle.tsx` | 아이콘 버튼 스타일 전환 |
| `src/components/ThemeToggle.tsx` | 신규 — 3단 순환, 저장 가드 |
| `src/components/Legend.tsx` | 신규 — 모드별 색상 범례 |
| `src/components/Fretboard.tsx` | 구조 불변, 노트 펄스 애니메이션 클래스만 추가 |

### 4.3 불변 영역
- `src/theory/`, `src/quiz/`, `src/audio/`, `src/lib/` 로직 전부.
- URL 상태 스키마(`viewUrl.ts`), 퀴즈 엔진/통계, 오디오.

### 4.4 에러/엣지 처리
- 테마 저장: 사운드 토글과 동일 패턴의 `localStorage` 차단 가드(try/catch).
- 시스템 테마 변경 실시간 반영은 "시스템" 모드일 때만.
- `prefers-reduced-motion` 전역 처리.

### 4.5 테스트 전략
- 기존 vitest 스위트 전부 통과가 기준. `data-testid`, `aria-*`, role 불변.
- 신규 테스트: ThemeToggle(순환/저장/차단 가드), Legend(모드별 항목 렌더링).
- 최종 검증: 브라우저에서 라이트/다크 × 4개 모드 실제 화면 확인.

### 4.6 진행 방식
점진적 마이그레이션, 단계마다 테스트 통과 후 커밋:
1. Tailwind 셋업 + 토큰/테마 기반(기존 CSS 공존)
2. 헤더(탭/테마 토글/사운드 토글)
3. 컨트롤 바
4. 스테이지(지판 카드 + 범례)
5. 퀴즈 모드
6. 구 CSS 제거 + 모바일/모션/접근성 마감
