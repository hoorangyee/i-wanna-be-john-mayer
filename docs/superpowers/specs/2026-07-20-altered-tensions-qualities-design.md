## 1. 요구사항

코드톤·오버레이 모드에서 다음을 추가로 확인할 수 있어야 한다.

사용자 결정:
1. **변형 텐션 b9/#9/#11/b13** — 도미넌트 퀄리티 전용.
2. **신규 퀄리티 3종** — diminished(dim, dim7), halfDiminished(m7b5), augmented(aug, aug7).
3. **13th 토글 정식 추가** — 모든 퀄리티에서 사용 가능한 자연 13(+9). b13은 도미넌트 전용 변형.
4. **텐션 UI는 플랫 필 자유 조합** — 기존 독립 표시 토글 철학 유지. 한 자리의 변형 둘 동시 켜기 허용 → `A7(b9,#9)` 얼터드 표현 가능.
5. **접근 방식 A(기존 모델 최소 확장)** — 퀄리티 3→6종, Extension 3→8종. 일반화 엔진 재작성(B)·코드 사전 복원(C)은 기각.

## 2. 이론 계층 (`src/theory/chords.ts`)

### 퀄리티

`ChordQuality`에 3종 추가. `QUALITIES` 순서: major, minor, dominant, diminished, halfDiminished, augmented. `QualityDef`에 `fifth: Tone` 필드 추가(기존 3종은 완전5도 `{interval: 7, degree: "5"}`).

| 퀄리티 | family | 3도 | 5도 | 7도(토글 시) | symbolBase / symbolWith7 |
|---|---|---|---|---|---|
| diminished | minor | b3 (+3) | b5 (+6) | bb7 (+9) | `dim` / `dim7` |
| halfDiminished | minor | b3 (+3) | b5 (+6) | b7 (+10) | `dim` / `m7b5` |
| augmented | major | 3 (+4) | #5 (+8) | b7 (+10) | `aug` / `aug7` |

- halfDiminished는 7 꺼짐 시 dim 트라이어드와 동일 표시 — dominant가 확장 없으면 메이저 트라이어드로 보이는 기존 전례와 같은 규칙(허용).
- 7th 종류는 퀄리티가 결정하는 기존 규칙 유지.

### 확장

```
Extension = "7" | "b9" | "9" | "#9" | "11" | "#11" | "13" | "b13"
EXTENSIONS 순서: 7, b9, 9, #9, 11, #11, 13, b13   (정규화·심볼 나열·URL 직렬화 순서)
```

- 인터벌/도수 라벨: b9 +1, 9 +2, #9 +3, 11 +5, #11 +6, 13 +9, b13 +8. 라벨은 토큰 그대로(`b9`, `#11` …).
- **변형 4종(b9/#9/#11/b13)은 도미넌트 전용.** 신규 헬퍼 `allowedExts(quality): readonly Extension[]` — 도미넌트는 8종 전부, 그 외는 자연 4종(7/9/11/13). 무효 조합 필터는 이 헬퍼를 단일 소스로 사용(UI 렌더·상태 전환·URL 파싱 공용).
- `normalizeExts`는 기존 동작(EXTENSIONS 순 정렬 + 중복 제거) 유지.

### 심볼 합성 (`chordSymbol`)

기존 규칙의 확장 — 나열 스타일 유지, 재즈 축약(`A13`) 안 함:
- 7 켜짐: `{symbolWith7}({uppers})` — 예: dominant+{7,b9,#9} → `7(b9,#9)`, dominant+{7,13} → `7(13)`.
- 7 꺼짐: `({addX,...})` — 변형에도 동일 적용: dominant+{b9} → `(addb9)`.
- 예: `Adim`, `Adim7`, `Am7b5`, `Aaug7`, `A7(b9,#9)`, `Amaj7(9,13)`.

### 피치클래스 충돌 가드 (`chordToneMap`)

dim의 bb7(+9)과 자연 13(+9)이 같은 피치클래스 — diminished+{7,13} 동시 선택 시 먼저 삽입된 톤이 승리하도록 `map.has(pc)` 가드 추가(삽입 순서상 bb7 라벨 유지). 다른 퀄리티·확장 조합에는 충돌 없음.

## 3. URL 상태 (`src/lib/viewUrl.ts`)

- `quality` 파라미터: TS 값 그대로 직렬화 (`quality=diminished|halfDiminished|augmented`) — `oneOf(QUALITIES)` 검증이 그대로 처리, 코덱 변경 제로.
- `ext` 파라미터: Extension 토큰 그대로 콤마 나열 (`ext=7,b9,#9`). `#`은 URLSearchParams가 `%23`으로 자동 인코딩 — 콤마도 이미 `%2C`로 인코딩되는 기존과 동일 성격, 별도 토큰 매핑(s9 등) 없음.
- **파싱 검증**: `ext` 파싱 후 `allowedExts(quality)`로 필터 — 비도미넌트 퀄리티에 변형 토큰이 오면 무시(드롭). 직렬화 측도 동일 필터로 방어적 정규화.
- 기본값 불변: `quality: "dominant"`, `exts: []`.
- 레거시 `chord` 파라미터 마이그레이션(maj/m/7/maj7/m7) 불변.

## 4. UI

- **컨트롤 바**: 퀄리티 셀렉트 6종(`QUALITY_NAMES` 표시명). 확장 필 그룹은 `allowedExts(quality)` 기반 렌더 —
  - 도미넌트: `[7th] [b9] [9th] [#9] [11th] [#11] [13th] [b13]` (8필)
  - 그 외: `[7th] [9th] [11th] [13th]`
  - 필 표기는 양 언어 동일: 자연은 `7th`·`9th`·`11th`·`13th`, 변형은 `b9`·`#9`·`#11`·`b13`. `aria-pressed` 등 기존 seg 패턴 유지.
  - 퀄리티를 도미넌트 → 다른 것으로 바꾸면 켜져 있던 변형 확장은 자동 드롭, 자연 확장은 유지.
- **타이틀**: 기존 메커니즘 불변 — `chordSymbol` 결과만 달라짐.
- **범례**: 슬롯 단위 유지 — 9 슬롯(9/b9/#9 중 하나라도 활성)이면 "9th/9도" 항목 하나를 슬롯 색으로 표시. 11 슬롯(11/#11), 13 슬롯(13/b13) 동일. `legendThirteenth`("13th"/"13도") 신규. 정확한 변형 표기는 프렛보드 도수 라벨이 담당. 트라이어드 범례(3rd/5th) 라벨 불변 — b5·#5도 5도 색.
- **색상**: 신규 토큰 `--tone-13` 라이트/다크 두 벌 — 기존 3(그린)/5(블루)/7(퍼플)/9(앰버)/11(틸)과 구분되는 로즈/마젠타 계열, 다크는 팔레트 대비 하한(3.2:1) 준수. Fretboard `TONE_FILL`에 `"13"` → `var(--tone-13)` 매핑 추가. `degreeFill`의 `degree.replace("b","")`를 `degree.replace(/[b#]/g,"")`로 일반화 — b9·#9·#11·b13·bb7·b5·#5 전부 해당 슬롯 색으로 귀결. 변형 전용 신규 색 없음(신규는 13 하나).
- **오버레이 모드**: 동일 모델 자동 적용(코드 레이어 = 루트+퀄리티+확장, 기존 결정 계승).
